What is Pocket Network?

A Decentralized RPC Provider

A Quick Background

In the Ethereum network, there’s two ways to broadcast a transaction:

  1. Run a client and broadcast directly to the network.
  2. Send a request to a node that offers RPC service.

Clients

  • mining nodes (in ETH 1.0)
  • validator nodes (in ETH 2.0)
  • full nodes (most recent 128 blocks)
  • archival nodes (all blocks)

Running a node means

  • Needing a machine that can run 24/7
    • A old computer?
    • A raspberry pi?
    • A rack server?
    • In the cloud?
  • Making sure you can access it securely.
  • Figuring out how to scale.
  • A lot of work unrelated to smart contract development.

But Wait!

  • Not here yet for web browsers.
  • Still needs to handle some complex behavior.
  • Considerable latency and bandwidth costs for historic lookups.

RPC

  • This is what MetaMask uses!
  • And Ledger’s Smart Contract Integration!
  • And Trust Wallet, and Gaurda, and MyEtherWallet…
  • And basically every web based dApp.

But What is RPC?

  • Remote Procedure Call.
  • Messages are communicated through JSON.
  • Nodes can expose an HTTP(S) API.
  • Provides the lowest technical barrier of entry for apps. Almost everything can communicate with JSON through HTTP(S)

Who Handles RPC?

The miners (soon to be validators)? Is this why I have to pay for gas? No.

Miners and validators can choose to serve RPC, but they have no obligation to do so.

There is no on-chain incentive for nodes to serve RPC, even though it's currently the primary way dApps interact with Ethereum.

Off Chain Solutions

Private entities stepped up to fill the need, ex:

  • Infura
  • Alchmey
  • Ankr

While accessible, they are all centralized and require monthly fiat payments to keep a dApp live.

Typically the only solution for smart contract developers who aren't prepared to run nodes.

… Until Pocket Network

Pocket Network

  • Is an independent blockchain that runs on POKT.
  • Node runners earn POKT for serving RPC relays.
  • Apps can stake POKT to get RPC access.
  • Provides a utilitarian market for blockchain infra.
  • Makes RPC permissionless and decentralized.

This all sounds neat, but....

How Does Pocket Help Developers?

Less Risks From Outages

Pocket Nodes run in a variety of configurations:

  • Self Hosted – physical machines at home or work
  • Cloud Hosting – AWS, Azure, Digital Ocean etc...
  • Rented Hardware – OVH, IBM Cloud Bare Metal etc...

This makes Pocket resilient to outages at the cloud provider and data center level.

Owning your Access

Once an app is staked, that relay rate is locked in.

As long as the app remains staked, there will be some guarantee of service.

This means that app developers:

  • Don’t have to worry about monthly billing.
  • Can’t accidentally run up a $100k compute bill.
  • Can recoup some of the initial infrastructure investment by unstaking and selling.

In Short

With Pocket Network, web3 app infrastructure becomes an investment, not a recurring cost.

Using Pocket

In JavaScript

ethers

import { ethers } from "ethers";

const URL = "https://eth-mainnet.gateway.pokt.network/...";
let web3 = new ethers.JsonRpcProvider(URL);

web3.getBlockNumber()
  .then(console.log);

web3.js

import { Web3 } from "web3";

const URL = "https://eth-mainnet.gateway.pokt.network/...";
let web3 = new Web3(URL);

web3.eth.getBlockNumber()
  .then(console.log);

Using Pocket

In Python

web3.py

from web3 import Web3

URL = "https://eth-mainnet.gateway.pokt.network/..."
w3 = Web3(Web3.HTTPProvider(URL))

print(w3.eth.get_block('latest'))

Cross Chain and Multi-Chain

The same application in the Portal can easily be used to get RPC service to multiple chains.

Eth and Polygon Example

import { ethers } from "ethers";

// These will end the same
let ETH  = "https://eth-mainnet.gateway.pokt.network/...";
let POLY = "https://poly-mainnet.gateway.pokt.network/..";

let ethWeb3 = new ethers.JsonRpcProvider(ETH);
let polyWeb3 = new ethers.JsonRpcProvider(POLY);

ethWeb3.getBlockNumber()
  .then(console.log);

polyWeb3.getBlockNumber()
  .then(console.log);

A full list of supported chains can be found at:

https://docs.pokt.network/home/resources/references/supported-blockchains
